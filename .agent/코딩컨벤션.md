## C 코딩 표준

### 1. 주석

#### 기본 원칙

1. 주석은 작성자 자신만이 아니라 다른 사람을 위한 것이다.
2. 자기만 이해할 수 있는 약어, 농담성 주석은 사용하지 않는다.
3. 주석은 항상 최신 상태를 유지한다.
4. 불필요한 중복 주석은 최소화한다.
5. Doxygen은 파일, 구조체, 함수, 멤버, 전역 정의 등 문서화가 필요한 항목에만 사용한다.
6. 무분별한 라인 단위 주석은 지양한다.
7. 기본 원칙은 Doxygen 스타일 주석 사용이다.

#### 주석 규칙

| 항목 | 규칙 |
| --- | --- |
| 기본 주석 | 기본 형식은 `/* 주석 내용 */` 사용 |
| 라인 주석 | `//` 사용 가능하지만, 특별한 사정이 없으면 블록 주석 권장 |
| 파일 설명 주석 | 각 소스 파일 최상단에 반드시 포함 |
| 함수 주석 | 간단 설명, 필요 시 긴 설명, `@param`, `@return` 포함 |
| struct / enum 주석 | 간단 설명과 필요 시 긴 설명, 멤버 설명 포함 |
| define / 전역 변수 | 선언 옆에 설명 주석 포함 |

#### 파일 설명 주석 예시

```c
/**
 * @file telnet.c
 * @brief Telnet main 함수
 *
 * 긴 설명은 한 줄을 넘긴 다음 넣어준다.
 * 하나, 둘, 셋
 * 넷, 다섯
 */
```

#### Main 페이지 주석 예시

```c
/**
 * @mainpage 첫페이지
 * @section intro 소개
 * 이 프로젝트는 telnet 프로그램이다.
 *
 * @section developer 개발자
 * 문병주(lovembj@monitorapp.com)
 *
 * @section history 역사
 * 이 프로젝트는 2010년 11월 시작되었다.
 */
```

#### 함수 주석 예시

```c
/**
 * @brief hello Main 함수
 *
 * 긴 설명은 한 줄을 넘긴 다음 넣어준다.
 * 하나, 둘, 셋
 * 넷, 다섯
 *
 * @param argc 인자 개수
 * @param argv 인자 문자열
 * @return 성공 여부
 */
```

#### struct / enum 주석 예시

```c
/**
 * @brief buffer 구조체
 * Telnet에서 전송되는 데이터를 처리하기 위한 입출력 버퍼 구조체
 */
struct buffer {
    char *buf;   /**< 데이터를 저장할 주소 공간 */
    int size;    /**< buf에 할당된 메모리 크기 */
    int head;    /**< buf에 저장된 데이터의 시작 인덱스 */
};

/**
 * @brief TRUE / FALSE 정의
 */
enum BOOLEAN {
    FALSE = 0,   /**< false */
    TRUE         /**< true */
};
```

#### define / 전역 변수 예시

```c
#define MAX_READ_BUF 1024 /**< 최대 read buffer 크기 */
short port;               /**< Telnet port 번호 */
```

### 2. Layout 및 Indentation

| 항목 | 규칙 |
| --- | --- |
| 라인 길이 | 한 줄은 80자 이내 권장 |
| 들여쓰기 | Space 4칸 기준 |
| 명령문 배치 | 한 줄에 하나의 수행 코드만 작성 |
| 함수 인자 | 콤마 뒤에는 공백 사용 |
| 함수 정의/호출 | 함수명과 괄호 사이에 공백 없음 |
| 괄호 표현식 | 괄호와 operand는 붙이고, operator는 공백으로 구분 |
| 조건문/반복문 | K&R 스타일 brace 사용 권장 |
| 단일 구문 조건문 | 단일 구문이어도 brace 사용 권장 |
| 조건문 비교 | Boolean이 아닌 경우 비교문을 명시적으로 작성 |
| 비교 순서 | 가능하면 `상수 == 변수` 형태 권장 |

### 3. Naming Rule

#### 기본 원칙

1. 이름만 보고도 의미를 유추할 수 있어야 한다.
2. 동일한 단어를 다른 의미로 재사용하지 않는다.
3. 구분자는 `_`를 사용한다.
4. 명사는 기본 단어 또는 약어를 사용한다.
5. 동사는 의미 전달을 위해 약어를 가급적 피한다.
6. 접두어는 약어 사용이 가능하다.
7. 모듈명은 해당 개체가 속한 모듈 약어를 반영한다.

#### 네이밍 규칙 표

| 구분 | 규칙 | 예시 |
| --- | --- | --- |
| 매크로 | 대문자 + `_` 사용 | `MAX_BUF_SIZE` |
| 상수 | 대문자 사용 | `MAX_NR_THREADS`, `GATEWAY_LOG_KEY_SIZE` |
| 타입 | `typedef` 타입은 `_t` 접미사 사용 | `u8_t`, `u16_t`, `fmem_t` |
| 지역 변수 | 소문자 사용, 필요 시 복수형 활용 | `sock`, `ip_groups` |
| file scope 변수 | 지역 변수 규칙 적용, `static` 선언 | `static int retry_count;` |
| 함수 | `[접두어]_[명사]_[동사]` 형태 권장 | `proxy_server_open`, `ai_list_free` |
| 전역 변수/함수 | 모듈명을 접두어로 사용 | `proxy_cfg_load` |

#### 타입 선언 예시

```c
struct _fmem {
    char *buf;
    int len;
};
typedef struct _fmem fmem_t;

typedef unsigned char u8_t;
typedef unsigned short u16_t;
```

## LOC Counting 표준

LOC 표준은 코딩 산출물인 코드를 정량화하기 위한 기준이다.

### 기본 룰

1. 기본적으로 하나의 statement는 하나의 라인으로 인정한다.
2. 각 구문은 특성에 맞추어 추가 룰을 적용한다.

### Statements 계산 규칙

| 구조 | 계산 규칙 | 비고 |
| --- | --- | --- |
| `if / else if / else` | 각각 1라인으로 계산 | 블록 내부 statement 별도 계산 |
| `switch` | `switch`, `case`, `default`, 내부 statement 각각 계산 |  |
| `while / do while` | 각각 1라인으로 계산 |  |
| `for` | 1라인으로 계산 | 내부 statement 별도 계산 |
| `return` | 각각 1라인 |  |
| `break / continue` | 각각 1라인 |  |
| `goto` | 각각 1라인 |  |
| `exit` | 각각 1라인 |  |
| NULL statement / blank | 무시 |  |
| 세미콜론으로 끝나는 statement | 각각 1라인 |  |
| braces | 실행 구문에 사용된 경우 1쌍을 1라인으로 계산 | 단순 블록 구분용 braces는 제외 가능 |
| `#define` | 각각 1라인 |  |
| `#include` | 각각 1라인 |  |
| `#if / #ifdef / #endif` | 각각 1라인 |  |
| 함수 선언/정의 | 각각 1라인 | 함수 본문은 별도 계산 |
| 변수 선언 | 각각 1라인 |  |
| 구조체 선언 | 각각 1라인 |  |
